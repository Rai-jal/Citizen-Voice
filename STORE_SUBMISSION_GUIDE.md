   - Linked to User: Yes (unless anonymous)

3. **Identifiers (Device ID, User ID)**

- None (unless you implement analytics)

**Data Linked to You:**
- Email address
- User-generated content (unless anonymous)

**Data Not Linked to You:**
- Usage analytics (if anonymized)
- Crash reports (if anonymized)

## üìã Submission Checklist

### Before Submission:

**Apple App Store:**
- [ ] All screenshots created and uploaded
- [ ] App description written and reviewed
- [ ] Privacy labels completed accurately
- [ ] Support URL configured
- [ ] Privacy Policy URL accessible
- [ ] Age rating completed
- [ ] App reviewed for compliance with App Store Guidelines

**Google Play Store:**
- [ ] Feature graphic created
- [ ] Phone screenshots created (2-8 images)
- [ ] App description written
- [ ] Data Safety section completed
- [ ] Content rating questionnaire completed
- [ ] Target audience defined
- [ ] App reviewed for compliance with Play Store policies

### Common Rejection Reasons to Avoid:

1. **Missing Privacy Policy** - Ensure it's accessible via URL
2. **Incomplete Data Safety** - Accurately declare all data collection
3. **Low-Quality Screenshots** - Use actual device screenshots, not mockups
4. **Misleading Descriptions** - Be accurate about app features
5. **Missing Permissions Justification** - Ensure all permissions are necessary and explained

## üöÄ Submission Steps

### Apple App Store:

1. Open App Store Connect
2. Create new app record
3. Upload build via Xcode or EAS Submit
4. Complete app information
5. Upload screenshots and metadata
6. Complete privacy labels
7. Submit for review

### Google Play Store:

1. Open Google Play Console
2. Create new app
3. Upload APK/AAB via Play Console or EAS Submit
4. Complete store listing
5. Complete Data Safety section
6. Complete content rating
7. Submit for review
